# Supabaseでメール確認を無効化する方法（開発環境推奨）

## 手順

### 1. Supabaseダッシュボードにアクセス
- URL: https://supabase.com/dashboard
- プロジェクトを選択: `fiertkuxlafeeqycywjh`

### 2. Authentication設定を開く
- 左メニューから **Authentication** をクリック
- **Providers** タブをクリック

### 3. Email設定を変更
- **Email** プロバイダーをクリック
- **Confirm email** トグルを **OFF** に変更
- **Save** ボタンをクリック

### 4. 既存ユーザーを確認済みにする（オプション）

SQL Editorで以下を実行：

```sql
-- すべての未確認ユーザーを確認済みにする
UPDATE auth.users
SET email_confirmed_at = NOW()
WHERE email_confirmed_at IS NULL;
```

### 5. アプリケーションを再起動

ブラウザでページをリロードして、再度ログインを試してください。

---

## 別の解決方法：ゲストとして新規登録

現在のログインがうまくいかない場合は、新しいアカウントで登録してください：

1. ログイン画面で「アカウントをお持ちでない方はこちら」をクリック
2. 新しいメールアドレス（テスト用）を入力
3. 「ゲストとして登録（1週間無料）」を選択
4. 登録後、すぐにアプリを使用できます

---

## トラブルシューティング

### エラー: "Invalid login credentials"
- パスワードが間違っている可能性があります
- 「アカウントをお持ちでない方はこちら」から新規登録してください

### エラー: "Email not confirmed"
- 上記の手順でメール確認を無効化してください
- または、SQL Editorで既存ユーザーを確認済みにしてください

### その他のエラー
- ブラウザのキャッシュをクリア
- シークレットモード/プライベートブラウジングで試す
- 別のブラウザで試す
